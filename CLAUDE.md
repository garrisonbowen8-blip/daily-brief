# Claude Code — Daily Brief Project Guidelines

## Autonomous session rules

These rules apply to ALL sessions in this directory, including autonomous runs.

### NEVER do these
- Commit or push `config.json` or `my-data.json` — these contain credentials
- Run `git push --force`
- Delete files outside this directory (`/Users/garrisonbowen/daily-brief/`)
- Make network requests other than: git push, ElevenLabs API, Google APIs
- Modify `daily-brief-data.js` directly — it is auto-generated by scripts/refresh.js
- Change the GitHub remote URL

### ALWAYS do these
- Verify `config.json` and `my-data.json` are in `.gitignore` before any `git add`
- Test scripts locally before committing
- Keep commits small and descriptive
- Preserve the existing liquid-glass visual design — no unsolicited redesigns
- Keep `daily-brief-voice.sh` and `daily-brief-voice.js` working for Garrison's existing setup

### Scope of autonomous runs
- Work only inside `/Users/garrisonbowen/daily-brief/`
- Allowed tools: Read, Edit, Write, Bash (scoped), Glob, Grep
- Bash commands allowed: node, npm, git (no force), bash scripts within this repo
- Do NOT install global npm packages without explicit instruction

### Project overview
This is a personal dashboard + JARVIS voice briefing system being packaged for open-source release.
- `index.html` — the dashboard (liquid-glass UI)
- `scripts/setup.sh` — setup wizard for new users
- `scripts/refresh.js` — standalone Google Calendar/Gmail refresh (no Claude dependency)
- `scripts/google-auth.js` — one-time OAuth flow
- `daily-brief-voice.js` — JARVIS spoken briefing script
- `daily-brief-voice.sh` — shell wrapper: ElevenLabs → afplay → iMessage

### Definition of done for the packaging build
- [ ] `npm install` runs without errors
- [ ] `scripts/setup.sh` is executable and has no syntax errors (`bash -n scripts/setup.sh`)
- [ ] `config.example.json` and `my-data.example.json` exist and are valid JSON
- [ ] `config.json` and `my-data.json` are gitignored
- [ ] README covers: clone → setup → Google OAuth → ElevenLabs → deploy → daily workflow
- [ ] All new files committed on a branch (not main)
